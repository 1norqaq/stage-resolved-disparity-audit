# Data Availability Statement

The empirical results in the paper are based on a linked application-level table reconstructed from a production applicant-tracking-system export. The table contains personal data from job applicants, candidate-job application histories, inferred protected attributes, model-derived representations, and recruitment outcomes. These data are not publicly released.

## Publicly released materials

The repository releases:

1. analysis code;
2. the expected table schema and codebook;
3. documentation of the analytic workflow;
4. scripts for generating a synthetic schema-compatible dataset;
5. templates for reproducing every table and figure once the governed table is available.

## Governed materials not released publicly

The following materials are not distributed:

- names, CV text, birth dates, geolocation, and contact information;
- raw status-change logs;
- raw profile and job embedding exports;
- employer, recruiter, and applicant identifiers;
- raw candidate-job score exports;
- linkage keys used to reconstruct the application-level table;
- any row-level data containing real applicant records.

## Access conditions

Replication using the original data requires an approved data-use agreement with the data controller. At minimum, access should require:

- documented research purpose;
- ethics or institutional review where applicable;
- secure storage and access logging;
- prohibition on re-identification or contact with applicants;
- prohibition on redistribution of row-level data;
- disclosure review before release of any aggregate table with small cells;
- deletion or return of governed files at the end of the access period.

## Synthetic data and smoke outputs

The synthetic data generated by `scripts/make_synthetic_data.py` matches the column names and approximate workflow structure needed to exercise the code. It is statistically artificial and must not be used to validate the empirical claims in the paper.

For double-anonymous review, `make smoke` and `make demo` use `scripts/smoke_test.py` to create a fast, robust synthetic schema check. Those outputs intentionally use `synthetic_smoke_*` or placeholder method labels for sparse downstream quantities. They verify output-file presence and schemas only; they do not reproduce the reported odds ratios, confidence intervals, mediation shares, or representation-probe values.
